# PRISM-R α/β算出方法 — 2方式の詳細比較

## 目的
PRISM-Rの銘柄選定に使用するα/βの算出について、自前実装（OLS-R）とstock-themes.com提供データ（OLS-ST）の2方式をMECEに比較し、どちらがPRISM-Rに適しているかを評価する。

---

## 1. 共通の回帰モデル構造

両方式とも、以下の単因子OLS回帰を基礎とする:

```
r_i(d) = α + β × r_theme(d) + ε(d)

r_i(d):     銘柄iの日dの日次リターン
r_theme(d): テーマ平均リターン（定義は方式により異なる）
α:          切片 = テーマ非連動の超過リターン（残差α）
β:          傾き = テーマ連動度
ε:          誤差項
R²:         決定係数 = テーマリターンによる説明力
```

---

## 2. OLS-R（PRISM-R 自前実装）の詳細仕様

### 2-1. 説明変数: 自己除外テーマ平均
```python
r_theme_ex_self(d) = Σ_{j≠i} r_j(d) / (n_members - 1)
```
- 銘柄i自身を**除外**した残りのメンバーの等ウェイト日次リターン平均
- **確認方法**: backtest_extended.py の panel['theme_ex_self'] で実装

### 2-2. 推定窓
- **固定63営業日**（約3ヶ月）ローリング
- リバランスごとに再計算（20営業日間隔）

### 2-3. 最小データ要件
- 63日中10日以上の有効ペア（y, x両方が有限値）

### 2-4. 出力値
```python
def ols_ab(y, x):
    # ... OLS計算 ...
    return a * n, b, r2
    # a * n = 日次αの累積値（= α_daily × 有効データ点数）
    # b = β（テーマ連動度）
    # r2 = 決定係数
```
- **α出力**: `α_daily × n`（累積値）。日次αではない点に注意。
- **統計的信頼性指標**: なし（t値/p値/SE未計算）

### 2-5. 信頼性調整: shrinkage関数
```python
def shrink_r2(r2):
    if r2 < 0.10: return r2 * 2        # 強い縮小
    if r2 <= 0.50: return 0.20 + (r2 - 0.10) * 2.0  # 線形補間
    return 1.0                          # 縮小なし
```
- R²をαの信頼性代理指標として使用
- hard filter（α>0, R²≥0.20等）の代替として設計
- 平均4.7銘柄/テーマの小ユニバースで候補枯渇を防ぐ

### 2-6. 最終スコア
```
score = α63_cumulative × shrink(R²_63)
```
- 各テーマからscore最大の1銘柄を選定

### 2-7. 自己除外の理論的根拠
- テーマ平均に銘柄i自身を含めると、y（銘柄リターン）がx（テーマ平均）の構成要素になる
- 特にn_members=4〜5の小テーマでは、銘柄iが平均の20〜25%を占める
- これによりβとR²が機械的に膨張し、αが過小評価される
- 自己除外することで、βは「他の銘柄群の動きに対する連動度」を純粋に測る

## 3. OLS-ST（stock-themes.com 提供データ）の詳細仕様

### 3-1. 説明変数: テーマ全体平均（自己除外なし）
```
r_theme(d) = Σ_j r_j(d) / n_members
```
- 銘柄i自身を**含む**全メンバーの等ウェイト日次リターン平均
- **確認方法**: APIレスポンスのtheme_returnが同一テーマ内の全銘柄で同一値であることを実測して確認済み
  - battery-storage: EOSE/FLNC/GWH/STEM 全て theme_return = -0.4056 (3M)
  - ai-semiconductor-core: AMD/AVGO/MRVL/MU/NVDA 全て theme_return = 0.2017 (3M)
  - semi-equip-front: ACLS/AMAT/ASML/KLAC/LRCX 全て theme_return = 0.2984 (3M)

### 3-2. 推定窓
- **7期間を並列提供**: 5D / 10D / 1M(≈21日) / 2M(≈42日) / 3M(≈63日) / 6M(≈126日) / 12M(≈252日)
- ユーザーが期間を選択可能

### 3-3. 出力値（検算済み）
```
alpha:            日次α（回帰切片）
alpha_ann:        年率換算α = alpha × 252  ← 検算一致
alpha_tval:       αのt統計量
alpha_pval:       αのp値（両側検定）
beta:             テーマ連動度（傾き）
r2:               決定係数
theme_return:     テーマ累積リターン（期間中）
theme_factor:     テーマ寄与 = β × theme_return  ← 検算一致
individual_factor: 銘柄固有リターン = 銘柄全体リターン − theme_factor
```

### 3-4. リターン分解の構造
```
銘柄iの期間リターン = theme_factor + individual_factor
                    = (β × r_theme_cumulative) + (total_return − β × r_theme_cumulative)

注: individual_factor ≠ α × 日数
    individual_factorはα寄与だけでなく、ε（残差）の累積も含む
```

### 3-5. 統計的信頼性指標
- **t値**: αの推定精度を反映
- **p値**: αの統計的有意性（p<0.05で5%有意水準）
- これらはPRISM-Rのshrinkage関数にはない情報

## 4. 差異の網羅的比較（MECE）

| 比較軸 | OLS-R（PRISM-R自前） | OLS-ST（stock-themes.com） |
|---|---|---|
| **回帰モデル** | 単因子OLS (同一) | 単因子OLS (同一) |
| **説明変数** | テーマ平均 **自己除外** | テーマ平均 **自己含む** |
| **推定窓** | 63日固定 | 7期間選択可能 |
| **α出力形式** | 累積値 (α_daily × n) | 日次 + 年率換算 |
| **t値/p値** | **なし** | **あり** |
| **R²** | あり（shrinkageの入力） | あり（そのまま表示） |
| **信頼性調整** | shrink(R²) 区分線形3段階 | なし（p値で代替可能） |
| **リターン分解** | なし | theme_factor + individual_factor |
| **データ更新** | CI/CDで毎日自動 | premium API依存 |
| **データソース** | yfinance日次価格 | stock-themes.com独自 |
| **テーマ構成** | frozen membership (207テーマ) | サイト管理者定義 (209テーマ) |
| **最小データ要件** | 63日中10日以上 | 不明 |
| **計算場所** | ローカル/GitHub Actions | サーバーサイド |

## 5. 自己除外の影響 — 定量的分析

### 5-1. 理論的影響
```
n_members = 5 の場合:

自己含むテーマ平均: r_theme = (r_i + Σ_{j≠i} r_j) / 5
  → r_i が r_theme の 20% を構成
  → OLSでは y の一部が x に混入
  → β が 1.0 方向に膨張（自己相関）
  → R² が過大評価
  → α が圧縮（テーマ平均に吸収される）

自己除外テーマ平均: r_theme_ex = Σ_{j≠i} r_j / 4
  → r_i と r_theme_ex は独立（iの情報は x に入らない）
  → β は純粋な連動度
  → R² は「他の銘柄による説明力」
  → α は「テーマ共通ファクターで説明できない固有リターン」
```

### 5-2. 実測例（EOSE, battery-storage, 3M）
```
OLS-ST (自己含む):   β=1.465  R²=0.6056  α=0.001067/day
OLS-R  (自己除外):   直接比較不可（frozen membershipの差異あり）

定性的予測:
  自己除外により β は低下（テーマ平均の分散が小さくなるため）
  自己除外により R² は低下（説明変数から自己成分が消えるため）
  自己除外により α は拡大（テーマ平均に吸収されていた固有成分が露出）
```

## 6. 各方式の長所・短所

### OLS-R（PRISM-R自前）
```
長所:
  ✅ 自己除外により推定値の統計的汚染が少ない
  ✅ 固定窓（63日）でパラメータ選択の自由度が低い → 過剰適合リスクが低い
  ✅ CI/CDで完全自動更新可能（外部依存なし）
  ✅ frozen membershipと完全に整合

短所:
  ❌ t値/p値がない → αの統計的有意性を直接判定できない
  ❌ shrinkageがR²のみに依存 → α自体の推定不確実性（SE(α)）を見ていない
  ❌ リターン分解がない → ポートフォリオ帰属分析に使えない
  ❌ 窓が63日固定 → regime変化への適応が遅い
```

### OLS-ST（stock-themes.com）
```
長所:
  ✅ t値/p値を提供 → αの統計的信頼性を直接評価可能
  ✅ 7期間を並列提供 → 複数時間軸でのregime分析が可能
  ✅ リターン分解（theme_factor + individual_factor）→ 帰属分析に有用
  ✅ alpha_annで年率換算済み → 異なる期間間の比較が容易

短所:
  ❌ 自己除外なし → βとR²が機械的に膨張（小テーマで顕著）
  ❌ 7期間選択可能 → 「都合の良い期間を選ぶ」過剰適合リスク
  ❌ premium API依存 → 自動更新が困難
  ❌ テーマ構成がサイト管理者定義 → frozen membershipとの差異
  ❌ αが過小評価される傾向（自己成分がテーマ平均に吸収）
```

## 7. PRISM-Rの銘柄選定スコアへの適合性評価

### 7-1. スコア計算の要件
```
PRISM-Rのスコア = α63 × shrink(R²_63)

必要な性質:
  (a) αが「テーマ平均を超える構造的な固有リターン」を正確に測る
  (b) R²が「αの推定信頼度」の妥当な代理指標である
  (c) 小ユニバース（平均4.7銘柄/テーマ）で安定して機能する
  (d) パラメータ数が少なく過剰適合しにくい
```

### 7-2. 各方式の適合度
```
要件(a) α精度:
  OLS-R: ✅ 自己除外により純粋な固有リターンを測定
  OLS-ST: ⚠ 自己含みによりαが圧縮される

要件(b) R²の意味:
  OLS-R: ✅ R² = 「他の銘柄の動きでどれだけ説明できるか」
  OLS-ST: ⚠ R² = 「自分自身を含むテーマ平均でどれだけ説明できるか」→ 膨張

要件(c) 小ユニバース安定性:
  OLS-R: ✅ 自己除外で4銘柄→3銘柄ベースだが、shrinkageで補正
  OLS-ST: ⚠ 自己含みで膨張した値が小テーマほど歪む

要件(d) 過剰適合リスク:
  OLS-R: ✅ 窓63日固定、期間選択なし
  OLS-ST: ⚠ 7期間から「最も都合の良い」期間を選ぶ誘惑
```

## 8. 今後の改善可能性

### 8-1. OLS-Rの改善候補（v2研究課題）
```
優先度1: score = α63 / residual_vol × shrink(R²)
  → raw αの「高ボラ銘柄偏重」問題を直接解決
  → ChatGPT Doc 13/15で推奨済み

優先度2: score = t(α63) × shrink(R²)
  → αの統計的有意性を組み込む
  → OLS-STのp値情報を参考にできる

優先度3: shrink(R², p_alpha) への拡張
  → R²だけでなくαのp値も縮小に使う
  → OLS-STデータをクロスチェック用に利用
```

### 8-2. 両方式の併用案
```
銘柄選定スコア: OLS-R（自己除外+固定窓で汚染が少ない）
外部検証:       OLS-STデータとのクロスチェック
  → PRISM-Rで選定した銘柄のαがOLS-STでもp<0.05で有意か確認
  → 一致すれば信頼度向上、不一致なら要注意フラグ
```
